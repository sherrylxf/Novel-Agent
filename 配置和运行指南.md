# Novel Agent 配置和运行指南

## 📋 前置要求

### 1. 环境要求

**方式A：本地开发环境**
- **JDK**: 17+
- **Maven**: 3.6+
- **MySQL**: 8.0+
- **PostgreSQL**: 14+ (with pgvector extension) - 可选
- **Neo4j**: 5.0+ - 可选

**方式B：WSL2 + Docker（推荐）**
- **Windows 10/11** with WSL2
- **Docker Desktop** for Windows（已启用WSL2集成）
- **LLM API Key**（OpenAI/DeepSeek/Qwen等）

### 2. IDE要求
- IntelliJ IDEA 2021.3+ 或 Eclipse 2021-12+
- 安装 Lombok 插件

### 3. LLM API配置
本项目**不使用本地模型**，通过API调用外部LLM服务。支持的API提供商：
- **OpenAI**: https://api.openai.com
- **DeepSeek**: https://api.deepseek.com
- **Qwen（通义千问）**: https://dashscope.aliyuncs.com/compatible-mode/v1
- **其他兼容OpenAI API的提供商**

## 🐳 WSL2 + Docker 快速启动（推荐）

### 前置步骤

1. **确保WSL2和Docker已安装**
```bash
# 在Windows PowerShell中检查WSL2
wsl --list --verbose

# 在WSL2中检查Docker
docker --version
docker-compose --version
```

2. **获取LLM API Key**
   - OpenAI: https://platform.openai.com/api-keys
   - DeepSeek: https://platform.deepseek.com/api_keys
   - Qwen: https://dashscope.console.aliyun.com/apiKey

### 步骤1：构建应用镜像

```bash
# 在项目根目录执行
cd /mnt/d/destop/Agent\ Study/Novel\ Agent

# 先编译项目
mvn clean package -DskipTests

# 构建Docker镜像
docker build -t novel-agent:1.0-SNAPSHOT -f ai-agent-station-study-app/Dockerfile ai-agent-station-study-app/

# 查看镜像
docker images | grep novel-agent
```

### 步骤2：启动所有服务（统一项目：novel-dev）

在WSL2中进入项目目录：

```bash
cd docs/dev-ops

# 使用项目名称 novel-dev 启动所有服务（数据库+应用）
docker-compose -p novel-dev --env-file ../../.env up -d

# 查看服务状态
docker-compose -p novel-dev ps

# 查看日志
docker-compose -p novel-dev logs -f
```

**服务端口映射：**
- **应用**: `localhost:8091`
- MySQL: `localhost:13306`
- PostgreSQL: `localhost:15432`
- Neo4j HTTP: `http://localhost:17474`
- Neo4j Bolt: `localhost:17687`
- phpMyAdmin: `http://localhost:8899`
- pgAdmin: `http://localhost:5050`

**所有服务都在 `novel-dev` 项目中，使用统一的网络 `novel-dev_novel-network`**

### 步骤3：配置LLM API

创建环境变量文件 `.env`（在项目根目录）：

```bash
# 在项目根目录创建.env文件
cat > .env << EOF
# LLM API配置
OPENAI_API_KEY=sk-your-api-key-here
OPENAI_BASE_URL=https://api.openai.com

# 或者使用DeepSeek
# OPENAI_API_KEY=sk-your-deepseek-key-here
# OPENAI_BASE_URL=https://api.deepseek.com

# 或者使用Qwen
# OPENAI_API_KEY=sk-your-qwen-key-here
# OPENAI_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
EOF
```

### 步骤4：验证运行

```bash
# 检查应用健康状态
curl http://localhost:8091/actuator/health

# 测试小说生成接口
curl -X POST http://localhost:8091/api/v1/novel/generate \
  -H "Content-Type: application/json" \
  -d '{
    "genre": "修仙",
    "coreConflict": "一个废材少年的逆袭之路",
    "worldSetting": "修真世界",
    "sessionId": "test-001",
    "maxStep": 5
  }'
```

### 常用Docker命令（统一项目：novel-dev）

```bash
cd docs/dev-ops

# 查看所有服务
docker-compose -p novel-dev ps

# 停止所有服务
docker-compose -p novel-dev down

# 停止并删除数据卷（谨慎使用）
docker-compose -p novel-dev down -v

# 重启应用
docker-compose -p novel-dev restart novel-agent-app

# 查看应用日志
docker-compose -p novel-dev logs -f --tail=100 novel-agent-app

# 查看所有服务日志
docker-compose -p novel-dev logs -f

# 进入应用容器
docker exec -it novel-dev-app bash

# 进入MySQL容器
docker exec -it novel-dev-mysql bash

# 更新应用（重新构建镜像后）
docker-compose -p novel-dev up -d --force-recreate novel-agent-app
```

## 🔧 配置步骤

### 步骤1：数据库初始化

#### 1.1 MySQL数据库初始化

```bash
# 登录MySQL
mysql -u root -p

# 创建数据库
CREATE DATABASE IF NOT EXISTS `novel_agent` DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_0900_ai_ci;

# 执行SQL脚本
source docs/dev-ops/mysql/sql/novel-agent.sql
```

或者直接执行：
```bash
mysql -u root -p < docs/dev-ops/mysql/sql/novel-agent.sql
```

#### 1.2 PostgreSQL数据库初始化（向量数据库）

```bash
# 登录PostgreSQL
psql -U postgres

# 创建数据库
CREATE DATABASE novel_vector;

# 连接到数据库
\c novel_vector

# 安装pgvector扩展
CREATE EXTENSION IF NOT EXISTS vector;
```

#### 1.3 Neo4j数据库（可选）

如果暂时不使用知识图谱功能，可以跳过Neo4j配置。

```bash
# 启动Neo4j（Docker方式）
docker run -d \
  --name neo4j \
  -p 7474:7474 -p 7687:7687 \
  -e NEO4J_AUTH=neo4j/password \
  neo4j:5.15.0
```

### 步骤2：修改配置文件

#### 2.1 修改 `application.yml` 或 `application-dev.yml`

根据你的实际环境修改数据库连接信息：

```yaml
spring:
  application:
    name: novel-agent
  
  # MySQL数据源配置（业务数据）
  datasource:
    mysql:
      driver-class-name: com.mysql.cj.jdbc.Driver
      url: jdbc:mysql://127.0.0.1:3306/novel_agent?useUnicode=true&characterEncoding=utf8&autoReconnect=true&zeroDateTimeBehavior=convertToNull&serverTimezone=UTC&useSSL=false
      username: root  # 修改为你的MySQL用户名
      password: 123456  # 修改为你的MySQL密码
      hikari:
        maximum-pool-size: 10
        minimum-idle: 5
        idle-timeout: 30000
        connection-timeout: 30000
        max-lifetime: 1800000
    
    # PostgreSQL数据源配置（向量数据库）
    pgvector:
      driver-class-name: org.postgresql.Driver
      url: jdbc:postgresql://127.0.0.1:5432/novel_vector
      username: postgres  # 修改为你的PostgreSQL用户名
      password: password  # 修改为你的PostgreSQL密码
      hikari:
        maximum-pool-size: 5
        minimum-idle: 2
        idle-timeout: 30000
        connection-timeout: 30000
  
  # Neo4j配置（知识图谱，可选）
  neo4j:
    uri: bolt://localhost:7687
    authentication:
      username: neo4j
      password: password  # 修改为你的Neo4j密码
  
  # AI配置（LLM API）
  ai:
    openai:
      api-key: ${OPENAI_API_KEY:your-api-key}  # 设置环境变量或直接填写
      base-url: ${OPENAI_BASE_URL:https://api.openai.com}
  
  # MyBatis配置
  mybatis:
    mapper-locations: classpath:/mybatis/mapper/**/*.xml
    config-location: classpath:/mybatis/config/mybatis-config.xml

# 线程池配置
thread:
  pool:
    executor:
      config:
        core-pool-size: 20
        max-pool-size: 50
        keep-alive-time: 5000
        block-queue-size: 5000
        policy: CallerRunsPolicy

# 日志配置
logging:
  level:
    cn.bugstack.novel: DEBUG
    org.springframework.ai: INFO
    root: INFO
```

#### 2.2 检查MyBatis配置文件

确保 `ai-agent-station-study-app/src/main/resources/mybatis/config/mybatis-config.xml` 存在：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE configuration PUBLIC "-//mybatis.org//DTD Config 3.0//EN" "http://mybatis.org/dtd/mybatis-3-config.dtd">
<configuration>
    <settings>
        <setting name="mapUnderscoreToCamelCase" value="true"/>
        <setting name="logImpl" value="STDOUT_LOGGING"/>
    </settings>
</configuration>
```

### 步骤3：检查依赖配置

#### 3.1 检查各模块的pom.xml

确保以下模块的pom.xml包含必要的依赖：

**ai-agent-station-study-app/pom.xml** 需要包含：
- Spring Boot Web Starter
- MyBatis Spring Boot Starter
- MySQL Connector
- PostgreSQL Driver
- Neo4j Driver（可选）
- Spring AI（可选，如果暂时不使用）

**ai-agent-station-study-infrastructure/pom.xml** 需要包含：
- MyBatis Spring Boot Starter
- Domain模块依赖

**ai-agent-station-study-domain/pom.xml** 需要包含：
- Types模块依赖

#### 3.2 添加缺失的依赖（如果需要）

如果app模块缺少Spring AI依赖，在 `ai-agent-station-study-app/pom.xml` 中添加：

```xml
<!-- Spring AI -->
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-postgresql-store</artifactId>
</dependency>
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-openai-spring-boot-starter</artifactId>
</dependency>

<!-- Neo4j Driver -->
<dependency>
    <groupId>org.neo4j.driver</groupId>
    <artifactId>neo4j-java-driver</artifactId>
</dependency>
```

### 步骤4：检查配置类

确保以下配置类存在：

1. **DataSourceConfig.java** - 多数据源配置
2. **Neo4jConfig.java** - Neo4j配置（可选）
3. **VectorStoreConfig.java** - 向量存储配置（可选）
4. **NovelAgentConfig.java** - Agent配置
5. **ThreadPoolConfig.java** - 线程池配置

## 🚀 运行步骤

### 方式1：使用IDE运行

1. **导入项目**
   - 使用IntelliJ IDEA打开项目根目录
   - 等待Maven依赖下载完成

cd ai-agent-station-study-app

## 🐛 常见问题排查

### 问题1：数据库连接失败

**错误信息：**
```
Could not get JDBC Connection; nested exception is java.sql.SQLException
```

**解决方案：**
1. 检查数据库是否启动
2. 检查连接信息是否正确（用户名、密码、端口）
3. 检查数据库是否已创建
4. 检查防火墙设置

### 问题2：MyBatis Mapper找不到

**错误信息：**
```
Could not find resource: mybatis/mapper/**/*.xml
```

**解决方案：**
1. 检查 `application.yml` 中的 `mybatis.mapper-locations` 配置
2. 确保Mapper XML文件在 `src/main/resources/mybatis/mapper/` 目录下
3. 检查文件命名是否正确

### 问题3：Neo4j连接失败

**错误信息：**
```
Unable to connect to neo4j
```

**解决方案：**
1. 如果暂时不使用Neo4j，可以在 `Application.java` 中排除 `Neo4jAutoConfiguration`
2. 或者启动Neo4j服务
3. 检查Neo4j连接信息是否正确

### 问题4：Spring AI依赖缺失

**错误信息：**
```
ClassNotFoundException: org.springframework.ai.vectorstore.VectorStore
```

**解决方案：**
1. 在 `ai-agent-station-study-app/pom.xml` 中添加Spring AI依赖
2. 如果暂时不使用RAG功能，可以注释掉相关代码

### 问题5：端口被占用

**错误信息：**
```
Port 8091 is already in use
```

**解决方案：**
1. 修改 `application.yml` 中的 `server.port`
2. 或者关闭占用端口的进程

## 📝 最小化运行配置（可选功能）

如果暂时不需要某些功能，可以简化配置：

### 1. 不使用Neo4j（知识图谱）

在 `Application.java` 中已经排除了 `Neo4jAutoConfiguration`，只需要：
- 注释掉 `Neo4jConfig.java`
- 注释掉 `Neo4jKnowledgeGraphService` 的使用

### 2. 不使用PostgreSQL（向量数据库）

- 注释掉 `VectorStoreConfig.java`
- 注释掉 `VectorRAGService` 的使用
- 在Agent中使用占位实现

### 3. 不使用Spring AI

- 注释掉Spring AI相关依赖
- 使用占位实现替代RAG功能
