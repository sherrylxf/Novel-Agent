# 用户可控流程使用指南

## 📋 概述

本系统实现了**用户可控制的Agent协作流程**，遵循"让用户掌管多个Agent来创作小说"的设计思想。每个Agent执行完成后，会暂停并等待用户审查和确认，用户同意后才继续执行下一个Agent。

## 🎯 设计理念

参考多Agent协作的最佳实践：
- **Top-down设计**：先规划整体流程，再执行具体步骤
- **用户掌控**：每个步骤都需要用户审查和确认
- **分步执行**：每次只执行一个Agent，避免一次性生成所有内容
- **可中断可恢复**：用户可以随时停止或继续流程

## 🔄 执行流程

```
1. 用户发起请求 → POST /api/v1/novel/generate
   ↓
2. RootExecuteNode（数据加载）
   ↓ [暂停，等待用户确认]
3. SeedExecuteNode（生成Seed）
   ↓ [暂停，等待用户确认]
4. PlanExecuteNode（规划小说）
   ↓ [暂停，等待用户确认]
5. VolumeExecuteNode（规划卷/册）
   ↓ [暂停，等待用户确认]
6. ChapterExecuteNode（生成章节梗概）
   ↓ [暂停，等待用户确认]
7. SceneExecuteNode（生成场景正文）
   ↓ [暂停，等待用户确认]
8. ValidationExecuteNode（校验）
   ↓
9. 完成
```

## 📡 API接口

### 1. 开始生成小说（SSE流式输出）

**接口**: `POST /api/v1/novel/generate`

**请求体**:
```json
{
  "genre": "都市重生",
  "coreConflict": "废柴少年的逆袭之路",
  "worldSetting": "草根大学生",
  "sessionId": "session_123456",
  "novelId": "novel-123",  // 可选，如果提供则继续生成
  "maxStep": 5  // 可选，最大执行步数
}
```

**响应格式（SSE）**:
```
data: {"type":"progress","sessionId":"session_123456","stage":"SEED","content":"节点[SeedExecuteNode]执行完成"}

data: {"type":"waiting_for_approval","sessionId":"session_123456","stage":"SEED","content":"节点[SeedExecuteNode]执行完成，等待用户确认后继续执行[NOVEL_PLAN]","data":{...生成的NovelSeed数据...}}

data: {"type":"complete","sessionId":"session_123456","novelId":"novel-123","content":"小说生成完成"}
```

### 2. 用户确认继续

**接口**: `POST /api/v1/novel/approve`

**请求参数**:
- `sessionId`: 会话ID（必填）
- `approved`: 是否同意继续（默认true）

**示例**:
```bash
POST /api/v1/novel/approve?sessionId=session_123456&approved=true
```

**响应**:
```json
{
  "type": "progress",
  "sessionId": "session_123456",
  "stage": "approval",
  "content": "已确认，继续执行"
}
```

## 💻 前端集成示例

### JavaScript/TypeScript示例

```javascript
// 1. 建立SSE连接
const eventSource = new EventSource('/api/v1/novel/generate', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    genre: '都市重生',
    coreConflict: '废柴少年的逆袭之路',
    worldSetting: '草根大学生',
    sessionId: 'session_' + Date.now(),
    maxStep: 5
  })
});

// 2. 监听消息
eventSource.onmessage = function(event) {
  const data = JSON.parse(event.data);
  
  switch(data.type) {
    case 'progress':
      // 显示进度
      console.log('进度:', data.stage, data.content);
      updateProgressUI(data.stage, data.content);
      break;
      
    case 'waiting_for_approval':
      // 显示生成的内容，等待用户确认
      console.log('等待确认:', data.stage, data.data);
      showApprovalDialog(data.stage, data.data, data.content);
      break;
      
    case 'complete':
      // 完成
      console.log('完成，小说ID:', data.novelId);
      eventSource.close();
      showCompleteMessage(data.novelId);
      break;
      
    case 'error':
      // 错误
      console.error('错误:', data.content);
      eventSource.close();
      showErrorMessage(data.content);
      break;
  }
};

// 3. 用户确认后调用确认API
function approveAndContinue(sessionId, approved) {
  fetch(`/api/v1/novel/approve?sessionId=${sessionId}&approved=${approved}`, {
    method: 'POST'
  })
  .then(response => response.json())
  .then(data => {
    console.log('确认结果:', data);
  })
  .catch(error => {
    console.error('确认失败:', error);
  });
}

// 4. 显示确认对话框
function showApprovalDialog(stage, data, message) {
  // 显示生成的内容
  const content = JSON.stringify(data, null, 2);
  
  // 创建确认对话框
  const dialog = document.createElement('div');
  dialog.innerHTML = `
    <h3>${message}</h3>
    <pre>${content}</pre>
    <button onclick="approveAndContinue('${sessionId}', true)">同意，继续</button>
    <button onclick="approveAndContinue('${sessionId}', false)">拒绝，停止</button>
  `;
  document.body.appendChild(dialog);
}
```

### React示例

```tsx
import { useState, useEffect } from 'react';

function NovelGeneration() {
  const [sessionId] = useState(`session_${Date.now()}`);
  const [currentStage, setCurrentStage] = useState('');
  const [generatedData, setGeneratedData] = useState<any>(null);
  const [waitingApproval, setWaitingApproval] = useState(false);
  
  useEffect(() => {
    // 建立SSE连接
    const eventSource = new EventSource('/api/v1/novel/generate', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        genre: '都市重生',
        coreConflict: '废柴少年的逆袭之路',
        worldSetting: '草根大学生',
        sessionId,
        maxStep: 5
      })
    });
    
    eventSource.onmessage = (event) => {
      const data = JSON.parse(event.data);
      
      switch(data.type) {
        case 'waiting_for_approval':
          setCurrentStage(data.stage);
          setGeneratedData(data.data);
          setWaitingApproval(true);
          break;
        case 'complete':
          setWaitingApproval(false);
          alert('生成完成！');
          break;
      }
    };
    
    return () => eventSource.close();
  }, [sessionId]);
  
  const handleApprove = async (approved: boolean) => {
    await fetch(`/api/v1/novel/approve?sessionId=${sessionId}&approved=${approved}`, {
      method: 'POST'
    });
    setWaitingApproval(false);
  };
  
  return (
    <div>
      {waitingApproval && (
        <div className="approval-dialog">
          <h3>当前阶段: {currentStage}</h3>
          <pre>{JSON.stringify(generatedData, null, 2)}</pre>
          <button onClick={() => handleApprove(true)}>同意，继续</button>
          <button onClick={() => handleApprove(false)}>拒绝，停止</button>
        </div>
      )}
    </div>
  );
}
```

## 🔧 核心组件说明

### 1. ExecutionStateService（执行状态管理服务）

管理sessionId到context的映射，支持暂停/恢复机制：
- `saveContext()`: 保存上下文
- `getContext()`: 获取上下文
- `createApprovalFuture()`: 创建等待确认的Future
- `approveAndContinue()`: 用户确认继续

### 2. NovelAgentOrchestrator（编排器）

支持分步执行：
- `initializeContext()`: 初始化上下文
- `executeNextStep()`: 执行下一步（每次只执行一个节点）

### 3. NovelGenerateResponseDTO（响应DTO）

新增消息类型：
- `waiting_for_approval`: 等待用户确认（包含生成的内容数据）

## 📝 注意事项

1. **SessionId管理**: 每个会话需要唯一的sessionId，用于关联上下文和状态
2. **超时处理**: 用户确认等待最多30分钟，超时后需要重新发起请求
3. **状态持久化**: 当前实现使用内存存储，重启后会丢失。生产环境建议使用Redis等持久化存储
4. **并发控制**: 每个sessionId使用独立的锁，支持并发请求

## 🚀 后续优化建议

1. **状态持久化**: 使用Redis存储执行状态，支持服务重启后恢复
2. **超时处理**: 添加超时机制，自动取消长时间未确认的请求
3. **批量确认**: 支持批量确认多个步骤
4. **回退机制**: 支持回退到上一个步骤重新生成
5. **预览功能**: 支持预览下一步将要生成的内容
