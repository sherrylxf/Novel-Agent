## 🎯 核心设计亮点

### 1. 多数据源配置
- **MySQL**：业务数据，使用MyBatis
- **PostgreSQL**：向量数据库，使用JdbcTemplate
- **Neo4j**：知识图谱，使用Spring Data Neo4j

### 2. 数据加载策略
- 策略模式：`ILoadDataStrategy`接口
- 并发查询：使用`CompletableFuture`
- 易于扩展：可以添加新的加载策略

### 3. 动态Bean注册
- 运行时注册：支持动态创建和注册Agent
- 线程安全：使用`synchronized`保证并发安全
- Bean管理：支持Bean的注册和移除

### 4. 责任链模式
- 节点化设计：每个节点负责一个执行步骤
- 上下文传递：通过`NovelContext`传递数据
- 易于扩展：可以添加新的执行节点

### 5. SSE流式输出
- 实时响应：流式输出执行进度
- 用户体验：提升交互体验
- 异步执行：使用线程池异步执行

## 📝 使用示例

### 1. HTTP接口调用

```bash
POST /api/v1/novel/generate
Content-Type: application/json

{
  "genre": "修仙",
  "coreConflict": "一个废材少年的逆袭之路",
  "worldSetting": "修真世界，分为炼气、筑基、金丹、元婴等境界",
  "sessionId": "session-123",
  "maxStep": 5
}
```

### 2. SSE响应格式

```
data: {"type":"progress","sessionId":"session-123","stage":"seed","content":"开始生成小说Seed..."}

data: {"type":"progress","sessionId":"session-123","stage":"plan","content":"小说规划完成"}

data: {"type":"complete","sessionId":"session-123","novelId":"novel-1234567890"}
```

### 3. 代码调用

```java
@Resource
private NovelAgentOrchestrator orchestrator;

// 执行完整流程
NovelPlan plan = orchestrator.generateNovel(
    "修仙",
    "一个废材少年的逆袭之路",
    "修真世界"

```
