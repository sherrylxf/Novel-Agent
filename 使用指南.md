# Novel Agent 使用指南

## 🚀 快速开始

### 1. 环境准备

**数据库：**
- MySQL 8.0+：业务数据存储
- PostgreSQL 14+ with pgvector：向量数据库
- Neo4j 5.0+：知识图谱

**Java环境：**
- JDK 17+
- Maven 3.6+

### 2. 数据库初始化

**MySQL：**
```bash
mysql -u root -p < docs/dev-ops/mysql/sql/novel-agent.sql
```

**PostgreSQL：**
```sql
-- 创建数据库
CREATE DATABASE novel_vector;

-- 安装pgvector扩展
CREATE EXTENSION IF NOT EXISTS vector;
```

**Neo4j：**
- 启动Neo4j服务
- 默认端口：7687

### 3. 配置修改

修改`application.yml`中的数据库连接信息：

```yaml
spring:
  datasource:
    mysql:
      url: jdbc:mysql://127.0.0.1:3306/novel_agent
      username: root
      password: 123456
    pgvector:
      url: jdbc:postgresql://127.0.0.1:5432/novel_vector
      username: postgres
      password: password
  neo4j:
    uri: bolt://localhost:7687
    authentication:
      username: neo4j
      password: password
```

### 4. 启动应用

```bash
mvn clean install
cd ai-agent-station-study-app
mvn spring-boot:run
```

## 📡 API接口使用

### 1. 生成小说（SSE流式输出）

**请求：**
```bash
POST http://localhost:8091/api/v1/novel/generate
Content-Type: application/json

{
  "genre": "修仙",
  "coreConflict": "一个废材少年的逆袭之路",
  "worldSetting": "修真世界，分为炼气、筑基、金丹、元婴等境界",
  "sessionId": "session-123",
  "maxStep": 5
}
```

**响应（SSE格式）：**
```
data: {"type":"progress","sessionId":"session-123","stage":"seed","content":"开始生成小说Seed..."}

data: {"type":"progress","sessionId":"session-123","stage":"plan","content":"小说规划完成"}

data: {"type":"complete","sessionId":"session-123","novelId":"novel-1234567890"}
```

### 2. 前端对接示例

**JavaScript：**
```javascript
const eventSource = new EventSource('/api/v1/novel/generate', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json'
    },
    body: JSON.stringify({
        genre: '修仙',
        coreConflict: '一个废材少年的逆袭之路',
        worldSetting: '修真世界',
        sessionId: 'session-123'
    })
});

eventSource.onmessage = function(event) {
    const data = JSON.parse(event.data);
    
    switch(data.type) {
        case 'progress':
            console.log('进度:', data.stage, data.content);
            break;
        case 'complete':
            console.log('完成，小说ID:', data.novelId);
            eventSource.close();
            break;
        case 'error':
            console.error('错误:', data.content);
            eventSource.close();
            break;
    }
};
```

## 🔄 执行流程

### 责任链执行流程

```
RootExecuteNode (数据加载)
    ↓
SeedExecuteNode (生成Seed)
    ↓
PlanExecuteNode (规划小说)
    ↓
VolumeExecuteNode (规划卷/册)
    ↓
ChapterExecuteNode (生成章节梗概)
    ↓
SceneExecuteNode (生成场景正文)
    ↓
ValidationExecuteNode (校验)
```

### 数据流转

```
NovelContext (上下文)
    ├─ novelId: "novel-123"
    ├─ currentStage: "seed"
    └─ attributes: {
         "genre": "修仙",
         "seed": NovelSeed,
         "plan": NovelPlan,
         ...
       }
```

## 🎯 核心功能说明

### 1. 多数据源配置

- **MySQL**：存储业务数据（小说、Seed、规划等）
- **PostgreSQL**：存储向量数据（RAG检索）
- **Neo4j**：存储知识图谱（人物、关系、伏笔）

### 2. 数据加载策略

- 使用`ILoadDataStrategy`接口
- 实现`NovelLoadDataStrategy`并发加载数据
- 支持扩展新的加载策略

### 3. 动态Bean注册

- 使用`AbstractNovelArmorySupport`提供动态注册功能
- 支持运行时创建和注册Agent
- 线程安全

### 4. 责任链模式

- 每个节点负责一个执行步骤
- 通过`NovelContext`传递数据
- 易于扩展新的执行节点

### 5. SSE流式输出

- 实时推送执行进度
- 提升用户体验
- 异步执行，不阻塞

## 📝 代码示例

### 1. 使用Orchestrator

```java
@Resource
private NovelAgentOrchestrator orchestrator;

NovelPlan plan = orchestrator.generateNovel(
    "修仙",
    "一个废材少年的逆袭之路",
    "修真世界"
);
```

### 2. 使用责任链

```java
@Resource
private RootExecuteNode rootExecuteNode;

NovelContext context = NovelContext.builder()
    .novelId("novel-123")
    .build();
context.setAttribute("genre", "修仙");
context.setAttribute("coreConflict", "废材逆袭");

rootExecuteNode.execute(context);
```

### 3. 使用数据加载策略

```java
@Resource
private Map<String, ILoadDataStrategy> loadDataStrategyMap;

ILoadDataStrategy strategy = loadDataStrategyMap.get("novelLoadDataStrategy");
strategy.loadData("novel-123", context);
```

## 🐛 常见问题

### 1. 数据源连接失败

**问题：** 启动时提示数据源连接失败

**解决：**
- 检查数据库是否启动
- 检查连接信息是否正确
- 检查数据库用户权限

### 2. Agent未注册

**问题：** 提示"Agent not found"

**解决：**
- 检查`NovelAgentConfig`是否正确注册Agent
- 检查Agent Bean是否创建成功

### 3. SSE连接断开

**问题：** 前端SSE连接立即断开

**解决：**
- 检查响应头设置是否正确
- 检查是否有异常抛出
- 检查超时设置
