## ⚠️ 部分应用的知识点

### 1. Agent-Prometheus监控场景（第3-14节）⚠️

**为什么需要监控：**
- ✅ **Agent执行监控**：监控各个Agent的执行时间、成功率、失败率，快速定位性能瓶颈
- ✅ **LLM调用监控**：监控LLM API调用延迟、token消耗、成本，优化调用策略
- ✅ **系统性能监控**：监控CPU、内存、数据库连接池使用情况，预防资源耗尽
- ✅ **任务进度监控**：监控小说生成任务的进度、完成率，提升用户体验
- ✅ **错误追踪**：监控异常和错误，快速定位和解决问题
- ✅ **业务指标监控**：监控生成字数、章节数、用户活跃度等业务指标

**现状：**
- ⚠️ 缺少监控基础设施（Prometheus + Grafana）
- ⚠️ 缺少Agent执行链路监控
- ⚠️ 缺少LLM调用性能监控
- ⚠️ 缺少系统资源监控
- ⚠️ 缺少业务指标监控

**需要实现：**
- Prometheus + Grafana监控体系
- Agent执行指标（执行时间、成功率、失败率等）
- LLM调用指标（延迟、token消耗、成本等）
- 系统资源指标（CPU、内存、数据库连接等）
- 业务指标（生成字数、章节数、任务完成率等）
- 告警机制（异常告警、性能告警）

**参考课程设计：**
```java
// 监控Agent执行
@Timed(name = "agent.execution", description = "Agent执行时间")
public void executeAgent(...) {
    // 记录指标
    meterRegistry.counter("agent.execution.count", "agent", agentType).increment();
    Timer.Sample sample = Timer.start(meterRegistry);
    try {
        // 执行Agent逻辑
    } finally {
        sample.stop(Timer.builder("agent.execution.time", "agent", agentType).register(meterRegistry));
    }
}
```
